## 1절 데이터 모델링의 이해
### 1. 모델링의 이해
- 모델링의 정의 : "당야한 현상을 표기법에 의해 표기하는 것"
- 3가지 특징 : 추싱화, 단순화, 명확화
- 3가지 관점 : 데이터 관점, 프로세스 관점, 상관관점


### 2. 데이터모댈의 기본 개념이해
- 데이터모델링의 정의 "정보시스템을 구축하기 위한 데이터관점의 업무분석기법"
- 데이터모델이 제공하는 기능 
  1. 시스템 가시화, 시스템 구조와 행동 명세화, 시스템 구축의 구조화된 틀 제공
  2. 문서화, 세부사항은 숨기는 다양한 관점 제공, 상세 수준의 표현방법 제공

### 3. 데이터모델링의 중요성 및 유의점
- 중요성
  1. 파급효과
  2. 복잡한 정보요구사힁의 간결한 표현
  3. 데이터 품질
- 유의점
  1. 중복 : 데이터베이스가 여러 장소에 같은 정보를 저장하지 않게 함
  2. 비유연성 : 데이터정의를 데이터 사용프로세스와 분리
  3. 비일관성 : 사용자가 처리하는 프러세와 혹은 관련 프로그램과 테이블을 독립적으로 하는 것

### 4. 데이터모델링의 3단계 진행
- 개념적 데이터모델링 : 추상화 수준이 높고 업무중심적이고 포괄적인 수준의 모델링 진향
- 논리적 데이터모델링 : 시스템으로 구축하고자하는 업무에 대해 키, 관계, 속성들을 정확하게 표현, 높은 재사용율
- 물리적 데이터모델링 : 실젤로 데이터베이스에에 이식할 수 있도록 성능, 저장 등 물리적인 성격을 고려해 솔계

### 5. 프로젝트 생명주기에서 데이터모델링
- 계획단계 -> 개념적모델링
- 분석단계 -> 논리적모델링  // 개념+논리적 모델링
- 설계단계 -> 물리적 모델링 // 물리적 모델링

### 6. 데이터 모델링에서 데이터독립성의 이해
 1. 데이터 독립성의 필요성: 유지보수비용 증가, 데이터복잡도 증가, 데이터중복성 증가, 요구사항대응 저하
 2. 데이터베이스 3단계구조 : 외부, 개념적, 내부
 3. 데이터독립성 요소 : 외부스키마, 개념스키마, 내부스키마
 4. 두 영역의 데이터독립성 : 논리적 독립성, 물리적 독립성
 5. MAPPING : "상호 독립적인 개념을 연결시켜주는 다리" // 외부적/개념적 사상 + 개념적/물리적 사상

### 7. 데이터모델링의 중요한 세가지 개념
- 3가지 요소
  1. 업무가 관여하는 어떤 것  (엔터티타입, 엔터티 / 엔터티, 인스턴스/어커런스)
  2. 어떤 것이 가지는 성격 (관계 / 페어링)
  3. 업무가 관여하는 어떤 것 간의 관계 (속성/ 속성값)

### 8. 데이터모델링의 이해관계자
1. 이해관계자의 데이터모델링 중요성 인식
2. 데이터모델링의 이해관계자

### 9.데이터모델의 표기접인 ERD의 이해
1. 엔터티 -> 사각형, 관계 -> 마름모 , 속성 -> 타원
2. ERD 모델링 방법 : 엔터티 그리기 -> 엔터티 배치 -> 관계설정 -> 관계명 -> 관계의 참여도 기술 -> 관계의 필수여부기술
